## 📦 新功能

### 1. Image Generation (图片生成)

OpenClaw 现在支持内置图片生成功能，可以直接通过 Agent 生成图片。无需调用外部图片生成服务，Agent 可以根据自然语言描述生成高质量图片。

**核心能力:**
- 文本到图片生成（Text-to-Image）
- 图片编辑与变体生成（Image-to-Image / Inpainting）
- 支持多种风格和尺寸

**支持的模型:** 
- OpenAI DALL-E 系列（`gpt-image-1`, `gpt-image-2`）
- Flux 系列（`flux-1.1-pro`, `flux-schnell`）
- Stable Diffusion（通过 HuggingFace Diffusers）

**使用方法:**
```markdown
用户: 生成一张 OpenClaw 架构图
Agent → 分析需求 → 调用 image_generate → 返回图片路径
```

**配置方式:**
```yaml
# openclaw.yaml
imageGeneration:
  primary: openai/gpt-image-2
  providers:
    openai/gpt-image-2:
      apiKey: ${OPENAI_API_KEY}
    stabilityai/stable-diffusion-xl:
      apiKey: ${STABILITY_API_KEY}
```

### 2. Subagent Timeout (子Agent超时控制)

新增子Agent运行超时配置，避免子任务无限期挂起。这是提高多Agent协作可靠性的关键功能。

**核心参数:**
| 参数 | 说明 | 默认值 |
|------|------|--------|
| `runTimeoutSeconds` | 子Agent最大运行时间（秒） | 300 |
| `cleanup` | 超时后清理策略 | delete |
| `mode` | 运行模式 | session |

**使用示例:**
```json
{
  "task": "分析 GitHub 最新 Issues 并生成报告",
  "runtime": "subagent",
  "runTimeoutSeconds": 180,
  "cleanup": "keep",
  "mode": "run"
}
```

**应用场景:**
- 限时数据采集任务（防止网络问题导致永久等待）
- 批量操作（限制每个子任务的执行时间）
- 并发控制（避免资源被长时间占用）

### 3. Local Embeddings (本地向量嵌入)

支持使用本地模型进行向量嵌入，降低 API 依赖和成本。适合对数据隐私有要求或需要大规模向量化的场景。

**支持的本地模型:**
- **Ollama embeddings** (`ollama/...`) — 推荐，支持主流嵌入模型
- **LocalAI embeddings** — 兼容 OpenAI API 格式
- **sentence-transformers** — 纯 Python 实现，无需 GPU

**配置示例:**
```yaml
# openclaw.yaml
embeddings:
  provider: local
  model: sentence-transformers/all-MiniLM-L6-v2
  
# 或使用 Ollama
embeddings:
  provider: ollama
  model: nomic-embed-text
  baseUrl: http://localhost:11434
```

**性能对比:**
| 方案 | 成本 | 延迟 | 隐私 | 适用场景 |
|------|------|------|------|----------|
| OpenAI API | $ | 低 | 数据离开本地 | 快速原型 |
| 本地 Ollama | $0 | 中 | 数据不出本地 | 生产环境 |
| sentence-transformers | $0 | 高 | 完全本地 | 极致隐私 |

## 🔒 安全更新

本次更新包含 **20+ 安全补丁**，建议所有用户立即更新。

**主要安全修复:**
- **工具调用权限绕过** — 修复了 `exec` 工具在特定条件下权限验证绕过的问题 (CVE-2026-xxxx)
- **路径遍历漏洞** — 加强了 `exec` 工具的路径验证，防止 `../` 路径遍历
- **API 密钥泄露** — 改进了 `${ENV_VAR}` 格式密钥的内存管理
- **XSS 风险** — 修复了 `browser` 工具在特定场景下的跨站脚本风险
- **Webhook 安全** — 增强了 Telegram/Discord Webhook 验证

## ⚠️ Breaking Changes

从旧版本升级时，请注意以下变更：

```yaml
# 旧配置 (v2026.4.x)
gateway:
  maxConcurrency: 10  # ❌ 已移除

# 新配置 (v2026.4.23)
gateway:
  poolSize: 10        # ✅ 使用此配置
  toolTimeout: 30    # ✅ 新增必需配置
```

**迁移步骤:**
1. 将 `maxConcurrency` 改为 `poolSize`
2. 添加 `toolTimeout: 30`（工具调用超时，单位：秒）
